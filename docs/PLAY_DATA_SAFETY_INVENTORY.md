# NearMeU Google Play Data Safety inventory

This is an implementation-grounded working sheet for the Google Play Data safety form. Re-check it whenever app behavior, Firebase configuration, SDK versions, or Play Console wording changes. The developer remains responsible for the final declaration.

## High-level answers

- **Does the app collect user data?** Yes.
- **Is collected data encrypted in transit?** Yes. NearMeU uses Firebase/Google HTTPS connections.
- **Can users request data deletion?** Yes, through the in-app Settings flow and the public account-deletion web resource.
- **Does the app sell user data?** No.
- **Does the app use data for third-party advertising?** No.
- **Does the app display the user's precise location to other users?** No. Exact coordinates are private; discovery uses rounded coordinates/cells and broad state/country information.
- **Is the app intended for children?** No. NearMeU is restricted to users aged 18 or older.

## Collection and sharing guidance

### Collection

Treat data as collected when it leaves the device and is stored or processed by NearMeU/Firebase. This includes profile data, exact and approximate location, messages, reports, photos, notification tokens, diagnostics and identifiers.

### Sharing

NearMeU uses Google/Firebase as service providers. Public profile fields and messages are transferred to other users as part of user-facing functionality. Review the current Play Console sharing exceptions carefully before answering the form; do not assume that a service-provider or user-initiated transfer must always be marked as sharing.

No data is transferred to advertising networks, data brokers, or unrelated marketing partners in V1.

## Required vs optional

- Google authentication, email, UID, nickname, age, gender and looking-for preference are required to create and use a complete NearMeU profile.
- Location permission is optional at the Android level, but nearby discovery cannot work correctly without location.
- Profile photo, chat messages, reports/descriptions and notifications are optional user actions or settings.
- Crashlytics, Analytics and Performance collection is enabled by default only in release builds; debug/profile builds are off unless explicitly opted in for controlled testing.

## Deletion behavior

Account deletion removes:

- Firebase Authentication account;
- public and private profile data;
- profile photo stored by NearMeU;
- exact/approximate location attached to the profile;
- notification tokens and preferences;
- block relationships;
- reports by or about the account;
- user-linked anti-abuse counters, report locks and moderation audit records.

Messages are hidden from the deleted account but may remain visible to the other chat participant. Firebase diagnostics/analytics records may remain for limited periods under provider retention controls.

## SDK review checklist

Before each production upload, review the current Firebase Android Play data-disclosure guidance for:

- Authentication and Google Sign-In;
- App Check / Play Integrity;
- Cloud Firestore;
- Cloud Functions;
- Cloud Storage;
- Cloud Messaging and Firebase Installations;
- Crashlytics and Firebase Sessions;
- Google Analytics for Firebase;
- Firebase Performance Monitoring.

Also compare `pubspec.lock` and Android transitive dependencies against the version of this inventory.
